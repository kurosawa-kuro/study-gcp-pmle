## 0. 前提

- **成果物**: PMLE（Google Cloud Professional Machine Learning Engineer）合格直結の**判断辞書型教材**（日本語 Markdown）。アプリ・DB・インフラは持たない。
- **対象試験**: 現行 PMLE。出題範囲・比率は公式試験ガイドに追従する（数値の正は `01_仕様と設計.md §1`）。
- **設計の正**: コンセプト・8 項目テンプレート・比較表・シグナル語・出題比率は `01_仕様と設計.md` を正とする。本書は重複を避け、必要時はそこを参照する。
- **スコープ境界**: 教材コンテンツ本体（`src/` の章原稿 + `doc/` の仕様）のみ。学習プラットフォーム・採点アプリ・実装ハンズオン教材は作らない（§5）。

## 5. やらないこと（スコープ外）

`01_仕様と設計.md §3.2` の破綻条件と整合させ、以下は明示的に作らない。

- **学習/模試プラットフォーム・採点アプリ** — 本書はコンテンツ教材であり、アプリ化はスコープ外。
- **実装ハンズオン教材化** — PMLE はコーディングを直接評価しない。手順書化に寄せると合格直結度が落ちる（破綻条件 4）。
- **サービス説明への偏り** — 使い方・画面操作の網羅説明はしない。「選ぶ/選ばない/ひっかけ」を主にする（破綻条件 1）。
- **本文と模擬問題の分離** — 章末問題集ではなく、本文中に判断材料を埋め込む（破綻条件 2）。
- **生成AIの後付け** — 旧来 ML だけで構成しない。Phase 3 で必ず含める（破綻条件 3）。
- **英語版・動画化** — 日本語 Markdown が canonical（`doc/README.md §4`）。多言語・動画は当面対象外。
