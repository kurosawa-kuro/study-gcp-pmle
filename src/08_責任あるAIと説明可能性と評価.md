# 第8章 責任あるAI・説明可能性・評価

> 意思決定: **予測の理由をどう説明し、公平性・バイアス・機密をどう扱い、モデルの良し悪しをどの指標で測るか。**
> Explainable AI（feature attribution）・Model Evaluation（指標・スライス）・公平性/バイアス・機密データ保護を切り分ける。思想⑤責任あるAI が核。「fairness / bias / explainability / privacy」が出たら**軽視する選択肢は必ず誤り**。

---

## §8.1 この章で切れる選択肢

- **Vertex Explainable AI** — feature attribution（予測理由）
  - **Sampled Shapley** / **Integrated Gradients** / **XRAI**（手法の使い分け）
- **Model Evaluation** — 指標 / スライス評価 / 公平性指標
- **公平性・バイアス対応** — データ/指標/スライスでの不公平の検出と是正
- **機密データ保護** — Sensitive Data Protection（旧 DLP）等での匿名化・最小権限

---

## §8.2 技術の最短定義

| 要素 | 一言定義 |
|---|---|
| Explainable AI | 各予測に対し、どの特徴がどれだけ寄与したか（feature attribution）を返す。 |
| Sampled Shapley | 微分不可能なモデル（決定木・アンサンブル等、表形式）向けの寄与度算出。 |
| Integrated Gradients | 微分可能なモデル（ニューラルネット）向けの寄与度算出。 |
| XRAI | 画像向け。寄与する領域（ピクセル群）をハイライトする。 |
| Model Evaluation | 精度 / AUC / RMSE 等の指標と、データのスライス別評価でモデルの良し悪しを測る。 |
| スライス評価 | 部分集団（性別・地域等）ごとに指標を比較し、不公平を検出する。 |
| Sensitive Data Protection | 機密・個人情報を検出・匿名化（マスキング/トークン化）する。 |

> ⚠ 要公式確認: attribution 手法とモデル種別の対応（Sampled Shapley / Integrated Gradients / XRAI の適用条件）、および機密保護サービスの名称（旧 Cloud DLP → Sensitive Data Protection）は変化し得る。現行仕様は公式ドキュメントで確認すること。

---

## §8.3 試験で選ぶ条件

### Explainable AI の手法選択

| 手法 | 選ぶ条件（決定打の語） |
|---|---|
| Sampled Shapley | **表形式 / ツリー・アンサンブル / 微分不可能**なモデル |
| Integrated Gradients | **ニューラルネット / 微分可能**なモデル（表形式・テキスト・画像の NN） |
| XRAI | **画像**で、寄与領域を見たい |

### その他

| テーマ | 選ぶ条件 | 試験キーワード |
|---|---|---|
| Model Evaluation | 指標で良し悪し / モデル比較 / スライス別の公平性 | `metrics`, `AUC/RMSE`, `sliced evaluation`, `compare models` |
| 公平性・バイアス | 部分集団で不公平 / 差別的影響を避けたい | `fairness`, `bias`, `disparate impact`, `protected attribute` |
| 機密保護 | 個人情報・機密の匿名化 / 最小権限 | `PII`, `de-identify`, `mask`, `DLP / Sensitive Data Protection` |

---

## §8.4 試験で選ばない条件

- **責任あるAI を無視しない**: `fairness` / `bias` / `privacy` / `explainability` が出ているのに、それに触れない選択肢は**ほぼ確実に誤り**（思想⑤）。
- **手法のミスマッチ**: ニューラルネットに Sampled Shapley を強制、ツリーモデルに Integrated Gradients を強制する選択肢は誤り。
- **XAI と Monitoring を取り違えない**: 「なぜこの予測か」は XAI、「分布がズレたか」は Monitoring（第7章）。
- **全体精度だけで判断しない**: 全体は良いが特定集団で悪い → スライス評価で検出すべき。

---

## §8.5 ひっかけパターン

1. **手法とモデル種別のミスマッチ** — 「ニューラルネットの説明」に Sampled Shapley、「ツリーモデルの説明」に Integrated Gradients を選ばせる。**微分可能か**で切る。画像なら XRAI。
2. **公平性の見落とし** — 全体精度が高い選択肢に飛びつかせるが、問題文に保護属性・公平性がある。→ スライス評価で部分集団を確認する選択肢が正解。
3. **機密データの扱い** — 個人情報をそのまま学習・ログ出力する選択肢。→ 匿名化（Sensitive Data Protection）・最小権限。
4. **XAI / Evaluation / Monitoring の三つ巴** — 「理由（XAI）」「良し悪し（Evaluation）」「ズレ検知（Monitoring）」を入れ替えて誘導する。問いの動詞で切る。

---

## §8.6 判断フローチャート

**説明可能性の手法選択**

```
何を説明したい？
 ├─ 画像のどこが効いたか → XRAI
 └─ 特徴量の寄与を知りたい → モデルは微分可能（NN）？
     ├─ Yes → Integrated Gradients
     └─ No（ツリー/アンサンブル/表形式） → Sampled Shapley
```

**責任あるAI の判断**

```
問題文に何が出ている？
 ├─ fairness / bias / 保護属性 → Model Evaluation のスライス評価で部分集団を確認・是正
 ├─ explainability / なぜ → Explainable AI（上の手法選択）
 ├─ PII / 機密 / privacy → Sensitive Data Protection で匿名化 + IAM 最小権限
 └─ どれだけ良いか（指標） → Model Evaluation
※ これらが出たら「無視する選択肢」は必ず除外する。
```

## §8.7 代表問題パターン

> 本書の自作シナリオ。

### 問題 1

> 表形式データで学習した**勾配ブースティング（ツリー系）**モデルについて、各予測でどの特徴が効いたかを説明したい。最適な手法は？
>
> A. Integrated Gradients
> B. **Sampled Shapley**
> C. XRAI
> D. TensorBoard

- **正解: B**。決定打は「**表形式**」「**ツリー系（微分不可能）**」。Sampled Shapley が適合。
- A を切る: Integrated Gradients は微分可能（NN）向け。
- C を切る: XRAI は画像向け。
- D を切る: TensorBoard は可視化で feature attribution ではない。

### 問題 2

> 融資審査モデルは全体精度が高いが、ある**保護属性（地域）**の部分集団で誤審率が著しく高いと指摘された。公平性の問題を検出・是正したい。最適な方法は？
>
> A. 全体精度をさらに上げる
> B. **Model Evaluation のスライス評価で部分集団ごとの指標を比較し、是正する**
> C. Explainable AI だけ有効化
> D. 監視を増やす

- **正解: B**。決定打は「**保護属性**」「**部分集団で誤審率が高い**」「**公平性**」（思想⑤）。スライス評価で集団差を検出。
- A を切る: 全体精度向上は集団差を解決しない（むしろ罠）。
- C を切る: XAI は理由説明で、公平性の測定・是正そのものではない（補助にはなる）。
- D を切る: 監視は本番のズレ検知で、公平性評価の主手段ではない。

### 問題 3

> ニューラルネットの**画像分類**で、誤分類の原因として「画像のどの領域が判断に効いたか」を可視化したい。最適な手法は？
>
> A. Sampled Shapley
> B. **XRAI**
> C. スライス評価
> D. Sensitive Data Protection

- **正解: B**。決定打は「**画像**」「**どの領域が効いたか**」。XRAI が領域ベースの寄与を示す。
- A を切る: Sampled Shapley は表形式・微分不可能モデル向け。
- C を切る: スライス評価は集団別指標で領域可視化ではない。
- D を切る: 機密保護は無関係。

---

## §8.8 最後に覚える一文

> **説明は「微分可能なら Integrated Gradients、ツリー/表形式なら Sampled Shapley、画像なら XRAI」。公平性はスライス評価、機密は匿名化＋最小権限。fairness/bias/privacy/explainability が出たら、無視する選択肢は必ず誤り。**
