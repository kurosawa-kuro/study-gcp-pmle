# 付録A シグナル語辞典（逆引き）

> 問題文の**表現から疑うべき選択肢を引く**ための逆引き索引。解くときの最初の一手に使う（序章 §0.4）。
> 出典は `doc/01_仕様と設計.md §2.3`。各章にも同じシグナル語を埋め込んでいるが、ここは横断で引けるよう統合した。

---

## A.1 使い方

1. 問題文から下表の「表現」に近い語を拾う。
2. 「疑うべき選択肢」を仮置きする。
3. 該当章の比較表（[付録B](91_付録_比較表.md)）と「ひっかけパターン（§N.5）」で確定/除外する。

> ⚠ シグナル語は**疑いの出発点**であって最終解ではない。必ず制約条件全体（レイテンシ・データ所在地・コスト・運用・公平性）と突き合わせること。

## A.2 モデル構築方式（→ [第1章](01_モデル構築方式の選択.md)）

| 問題文の表現 | 疑うべき選択肢 |

## A.4 サービング（→ [第3章](03_本番サービングの選択.md)）

| 問題文の表現 | 疑うべき選択肢 |
|---|---|
| real-time / low latency / synchronous / per-request / ユーザー対面 | Online prediction endpoint |
| nightly / scheduled / millions of records / asynchronous / offline scoring | Batch prediction |
| cold start を避けたい / レイテンシ保証必須 | min_replica >= 1（常駐） |
| トラフィック変動 / コスト最適化 | autoscaling（min/max replica） |
| 前処理・後処理を推論に含めたい / training-serving skew を防ぐ | カスタムコンテナ / CPR |
| VPC 内限定 / 公開不可 | private endpoint |
| 5% から徐々に切替 / 段階的比較 | traffic split（canary / A-B） |
| 応答は旧のまま新モデルを観測だけ | shadow deployment |
| 既存 GKE 資産 / scale-to-zero / 細かい制御 | GKE + KServe |

## A.8 モデル運用監視（→ [第7章](07_モデル運用監視の判断.md)）

| 問題文の表現 | 疑うべき選択肢 |
|---|---|
| デプロイ直後から学習時と挙動が違う / 前処理の不一致 | training-serving skew 検知（baseline=学習データ） |
| 時間経過で徐々に劣化 / 入力分布の変化 | prediction / feature drift 検知（baseline=過去本番） |
| 効いている特徴が変わった | feature attribution drift 検知 |
| 本番デプロイ後に継続的にアラート | Vertex AI Model Monitoring |
| 検知して再学習を起動 | Monitoring → 再学習パイプライン（CT、第4章） |

## A.10 生成AI活用・RAG（→ [第9章](09_生成AI活用とRAGの判断.md)）

## A.11 生成AIチューニング・評価（→ [第10章](10_生成AIチューニングと評価.md)）

| 問題文の表現 | 疑うべき選択肢 |
|---|---|
| まず低コストで改善 / few-shot | プロンプト設計 |
| 一貫した文体・出力形式・専門タスク適応 | Fine-tuning（SFT） |
| 品質を保ちコスト/レイテンシ削減 / 小型化 | 蒸留（distillation） |
| 2 案の優劣を比較 / 自動採点 | Gen AI 評価（pairwise / pointwise） |
| 最新/社内知識を反映（≠ fine-tuning） | RAG（第9章） |

## A.12 所在地・コスト・セキュリティ（→ [第11章](11_データ所在地とコストとセキュリティ.md)）

| 問題文の表現 | 疑うべき選択肢 |
|---|---|
| data residency / region に留める / regulatory | region 固定・データ所在地 |
| データ持ち出し（exfiltration）を防ぐ境界 | VPC Service Controls |
| 公開せず VPC 内限定 | private endpoint |
| 暗号鍵を自社管理 | CMEK（Cloud KMS） |
| 外部CI/別クラウドから安全接続 / 鍵を配らない | Workload Identity Federation |
| 過剰権限を避ける / 監査 | IAM 最小権限 |
| PII / 機密 / de-identify | Sensitive Data Protection（旧 DLP） |
| 即時不要・大量でコスト削減 | batch prediction |
| アイドル多い/変動大でコスト削減 | scale-to-zero / autoscaling |

> ⚠ 要公式確認: 生成AI系サービス名（Model Garden / Vertex AI Search / Agent Builder / RAG Engine / Gemini 世代）と機密保護サービス名（DLP → Sensitive Data Protection）はリブランド進行中。現行公式名は要確認。
